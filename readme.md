# Real-Time Multi-Road Adaptive Traffic Control System

A computer-vision-based adaptive traffic signal control system for **Prithivi Chowk**, the highest-traffic intersection in **Pokhara, Nepal**. The system uses YOLOv8s object detection and centroid tracking across four parallel video streams to dynamically compute Green Signal Time (GST) based on real-time traffic demand, with a Nepal free-left traffic rule multiplier.

## Features

- **4 parallel video processing threads** — YOLOv8s detection on each approach road simultaneously
- **Real-time vehicle counting** — CentroidTracker with direction filtering and counting lines
- **Dynamic GST computation** — Green Signal Time recalculated immediately before each signal switch using live queue data
- **Signal cycle** — A→B→C→D (China Pull → Airport → Sabhagriha Chowk → Naya Bazar) with orange preview
- **PyGame intersection simulation** — High-fidelity visualization with vehicle physics, lane-following, and collision avoidance
- **Live verification display** — 2×2 OpenCV grid showing real-time bounding boxes and count overlays
- **Data output** — CSV, TXT, and JSON logging on every signal switch

## Usage

### Real-time mode (live video + simulation)

```bash
python main.py
```

Launches 4 parallel video processing threads, dynamic signal controller, PyGame simulation, and 2×2 OpenCV verification window.

### Standalone simulation mode (cached data)

```bash
python simulation/main.py
```

Uses pre-computed data from `outputs/gst_cache.json` (generated by a prior `python main.py` or `python process_all_videos.py` run).

### Batch video processing (legacy)

```bash
python process_all_videos.py
```

Processes all videos sequentially and writes aggregated results to `outputs/`.

## Key Design Decisions

- **Single YOLO model with lock** — Prevents GPU OOM from 4 simultaneous instances
- **Daemon threads** — Auto-terminate when main window closes
- **Free-left rule** — ⅓ multiplier in GST formula; left-turn lane always open
- **Class-basis lane assignment** — Vehicles distributed evenly across lanes per vehicle type
- **Dynamic lane offsets** — Lane positions computed from per-approach lane counts
